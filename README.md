# Anime 365 Emby Sidecar

Этот репозиторий содержит код приложения, которое налаживает интеграцию между Anime 365 и вашим сервером Emby. Приложение называется сайдкаром (sidecar), потому что оно задумано так, чтобы работало рядом с сервером Emby. Сайдкар отслеживает ваш список тайтлов на Anime 365 и загружает в ваш Emby серии к просмотру. После просмотра серия на Anime 365 отмечается просмотренной. Просмотренные серии со временем удаляются из вашего Emby-сервера.

Обратите внимание, что для работы сайдкара необходим аккаунт Anime 365 с оплаченной премиум-подпиской.

## Подробнее об устройстве сайдкара

Это приложение, написанное на Go, и скомпилированное под все операционные системы и архитектуры процессоров. Предполагается, что оно будет запущено с помощью Docker, Docker Compose, но также можно запустить бинарный файл и напрямую. Идеальный сценарий - это когда у вас в одном Docker Compose запущены Emby и сайдкар

Создайте директорию, в которой будут храниться загруженные медиа. К этой директории одновременно должен быть доступ у сайдкара и у Emby. Назовите директорию "anime365".

В настройках вашего сервера Emby создайте новую Library с типом "TV shows". Назовите библиотеку "Anime 365". Выберите директорию, которую вы создали ранее. Зайдите в настройки этой библиотеки и поотключайте все галочки и переключатели, связанные с подгрузкой метаданных.

В настройках вашего сервера Emby выпустите API Key. Например, ключ можно назвать "Anime 365 Sidecar".

## Конфигурация

### `SIDECAR_ANIME365_BASE_URL`

Адрес Anime 365. Пример: `https://smotret-anime.app`.

### `SIDECAR_ANIME365_LOGIN`

Ваш логин от Anime 365. Пример: `foobar@example.com`.

### `SIDECAR_ANIME365_PASSWORD`

Ваш пароль от Anime 365. Пример: `qwerty`.

### `SIDECAR_SCAN_SOURCES`

Откуда брать сериалы для загрузки.

Возможные значения: `list_watching` (список "Смотрю"), `list_planned` (список "Запланировано"). Можно указать несколько через запятую. Можно также указать ID тайтлов. Например, `list_watching,list_planned` (списки "Смотрю" и "Запланировано") или `list_watching,8868` (список "Смотрю" и 1й сезон Гинтамы). По умолчанию имеет значение `list_watching`.

### `SIDECAR_SCAN_IDLE_INTERVAL`

Как часто делать обход по Anime 365 в поисках новых серий для загрузки. При старте сайдкара выполняется первый обход, и как только он завершится, сайдкар войдет в режим ожидания - до запуска следующего обхода. Этот параметр отвечает за то, как долго будет длиться этот режим ожидания.

Указывается в формате `{duration}s`, `{duration}m` или `{duration}h`. Примеры: `30s`, `15m`, `1h` и `1h30m`. По умолчанию имеет значение `5m`. Под капотом используется функция [`time.ParseDuration`](https://pkg.go.dev/time#ParseDuration), так что можно передать любую валидную строку.

### `SIDECAR_TRANSLATIONS`

Какие переводы нужно загружать. Указывается в формате `<язык>_<тип>`. Можно указать несколько через запятую.

- Язык указывается в виде двухбуквенного кода, например, `ru`, `en` или `ja`.
- Поддерживаемые типы: `subtitles` (субтитры), `dub` (дубляж или озвучка), `raw` (исходный файл).

По умолчанию имеет значение `ru_subtitles,ru_dub`.

Рекомендуется не указывать переводы, которые вам не нужны, так как это прямо пропорционально влияет на потребление диска. Например, если смотрите только в русских субтитрах, укажите `ru_subtitles`.

## Рекомендации по эксплуатации

Без необходимости не трогайте загруженные файлы вручную. Сайдкар полагается на название файлов и их расположение в директориях.

Сайдкар умеет работать только с одним аккаунтом Anime 365. При необходимости можно поднять несколько сайдкаров с разными аккаунтами. В таком случае не указывайте нескольким сайдкарам одинаковые директории, в которых будет храниться загруженное медиа, иначе их логика будет конфликтовать друг с другом. Укажите разные директории.

## Внутреннее устройство

Сайдкар - это приложение на Go. Это приложение параллельно делает несколько задач:

- Мониторит ваш список тайтлов на Anime 365.
- Качает на диск все еще не загруженные серии из тайтлов, которые встречаются в вашем списке.
- Обновляет метаданные у загруженных тайтлов и серий в вашем сервере Emby.

Каждая из этих задач выполняется в отдельной горутине.

Чтобы не создавать лишнюю нагрузку на Anime 365, Shikimori и Jikan (MyAnimeList), все задачи по обновлению метаданных, а также загрузка серий, выполняются синхронно. В один момент времени может качаться только одна серия (пусть и в несколько потоков), в один момент времени могут обновляться метаданные только одного тайтла и т.д.
