# Anime 365 Sidecar

Этот репозиторий содержит код приложения, которое налаживает интеграцию между Anime 365 и вашим сервером Emby. Приложение называется сайдкаром (sidecar), потому что оно задумано так, чтобы работало рядом с сервером Emby. Сайдкар отслеживает ваш список тайтлов на Anime 365 и загружает в ваш Emby серии к просмотру. После просмотра серия на Anime 365 отмечается просмотренной. Просмотренные серии со временем удаляются из вашего Emby-сервера.

Обратите внимание, что для работы сайдкара необходим аккаунт Anime 365 с оплаченной премиум-подпиской.

## Как запустить

Сперва нужно настроить сервер Emby.

Создайте директорию, в которой будут храниться загруженные медиа. К этой директории одновременно должен быть доступ у сайдкара и у Emby. Назовите директорию, например, "anime365".

В настройках вашего сервера Emby создайте новую Library с типом "TV shows". Назовите библиотеку "Anime 365". Выберите директорию, которую вы создали ранее. Зайдите в настройки этой библиотеки и поотключайте все галочки и переключатели, связанные с подгрузкой метаданных.

Теперь можно разворачивать сайдкар. Есть 2 способа установки: через Docker или скачать исполняемый файл и запустить его самостоятельно.

### Запуск через Docker

Если ваш сервер Emby уже поднят через Docker, то лучше всего будет так же поднять и сайдкар.

Пример конфигурации Docker Compose, где поднимается Emby вместе с сайдкаром, и у обоих сервисов указывается одинаковый волюм:

```yaml
services:
  emby:
    image: emby/embyserver:latest
    container_name: emby
    restart: unless-stopped
    pull_policy: weekly
    volumes:
      - ./emby/config:/config
      - ./emby/media/anime365:/mnt/anime365
    ports:
      - "8096:8096"
      - "8920:8920"
    environment:
      UID: "1000"
      GID: "1000"
      GIDLIST: "1000"
    healthcheck:
      test: ["CMD", "wget", "--quiet", "--output-file", "/dev/null", "http://localhost:8096/System/Ping"]
      interval: 30s
      timeout: 10s
      retries: 3

  sidecar:
    image: ghcr.io/flaksp/anime365-sidecar:latest
    container_name: sidecar
    restart: unless-stopped
    pull_policy: weekly
    volumes:
      - ./emby/media/anime365:/mnt/anime365
    user: "1000:1000"
    environment:
      SIDECAR_ANIME365_BASE_URL: https://smotret-anime.app
      SIDECAR_ANIME365_LOGIN: user@example.com
      SIDECAR_ANIME365_PASSWORD: qwerty
      SIDECAR_EMBY_API_KEY: d505e9041faf4447935aedd336224b05
      SIDECAR_EMBY_BASE_URL: http://emby:8096/emby
      SIDECAR_EMBY_LIBRARY_ID: 3
      SIDECAR_EMBY_USER_ID: 9283b8ae8bee4ccba61174e310ff0ab1
      SIDECAR_LIBRARY_DIRECTORY: /mnt/anime365
    depends_on:
      emby:
        condition: service_healthy
```

У сервиса `sidecar` вместо `:latest` можно использовать зафиксированную версию Docker-образа. Актуальную версию можно посмотреть в разделе [Packages](https://github.com/flaksp/anime365-sidecar/pkgs/container/anime365-sidecar).

### Запуск бинарного файла

Перейдите в раздел [Releases](https://github.com/flaksp/anime365-sidecar/releases) и у самого свежего релиза скачайте исполняемый файл для своей операционной системы. Вы можете запустить загруженный исполняемый файл просто так и он будет работать, но при закрытии приложения, перезапуске ПК или при непредвиденной ошибке сайдкар закроется - и придется запускать его по новой. Поэтому лучше всего сделать так, чтобы сайдкар всегда был запущен: стартовал при включении системы, и при ошибках чтобы перезапускался автоматически.

Docker Compose решает эту проблему за нас и это рекомендуемый способ эксплуатации на Linux. Если не хочется использовать Docker на Linux или у вас macOS, то не могу ничего порекомендовать из своего опыта, но скорее всего поиск в интернете или ChatGPT смогут помочь советом. На Windows можно попробовать [servy](https://github.com/aelassas/servy).

## Рекомендации по эксплуатации

Без необходимости не трогайте загруженные файлы вручную. Сайдкар полагается на название файлов и их расположение в директориях.

Сайдкар умеет работать только с одним аккаунтом Anime 365. При необходимости можно поднять несколько сайдкаров с разными аккаунтами. В таком случае не указывайте нескольким сайдкарам одинаковые директории, в которых будет храниться загруженное медиа, иначе их логика будет конфликтовать друг с другом. Укажите разные директории.

## Внутреннее устройство

Сайдкар - это приложение на Go. Это приложение параллельно делает несколько задач:

- Мониторит ваш список тайтлов на Anime 365.
- Качает на диск все еще не загруженные серии из тайтлов, которые встречаются в вашем списке.
- Обновляет метаданные у загруженных тайтлов и серий в вашем сервере Emby.

Каждая из этих задач выполняется в отдельной горутине.

Чтобы не создавать лишнюю нагрузку на Anime 365, Shikimori и Jikan (MyAnimeList), все задачи по обновлению метаданных, а также загрузка серий, выполняются синхронно. В один момент времени может качаться только одна серия (пусть и в несколько потоков), в один момент времени могут обновляться метаданные только одного тайтла и т.д. Это, конечно, все работает сильно дольше, чем могло бы быть, но не создает излишнюю нагрузку на внешние сервисы, что уменьшает вероятность получения бана по IP или ошибок рейт-лимитинга.

Первая загрузка всей библиотеки, в зависимости от ваших настроек сайдкара и размера ваших списков, может занять какое-то время.

## Вопросы и проблемы

### Планируется ли поддержка Plex?

В теории поддержка возможна, но пока таких планов нет.

### Планируется ли поддержка Jellyfin?

В Jellyfin есть проблемы с реализацией группировки разных версий одного эпизода. Такая группировка необходима для того, чтобы было возможно объединить разные переводы эпизода в один.
